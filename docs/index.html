How It Works

1

Lock

Deposit FLOW tokens with a Poseidon commitment. The commitment hides your secret, nullifier, and amount.

Flow EVM
2

Prove

Off-chain, build a Merkle tree from all deposits. Generate a Groth16 proof that you know a valid leaf without revealing which one.

Off-chain (snarkjs)
3

Verify

Submit the proof to Starknet. The garaga verifier checks the Groth16 proof on-chain in ~1977 felts of calldata.

Starknet
4

Claim

If the proof is valid and the nullifier hasn't been used, your shielded balance is credited. A relayer can submit on your behalf for a fee.

Privacy Guarantees

  • ✓
    Fixed denomination pools
    All deposits in a pool have the same amount. Eliminates amount-based transaction linking.
  • ✓
    On-chain Merkle tree
    Poseidon incremental tree computed on-chain (Flow EVM). Root history on Starknet. Trustlessly verifiable.
  • ✓
    Commitment hiding
    Poseidon(Poseidon(secret, nullifier), amount). On-chain data cannot reveal secret or link to withdrawal.
  • ✓
    No privacy-leaking events
    Mint events emit only nullifier hash and amount. No recipient, no off-chain identifiers.
  • ✓
    Relayer support
    Anyone can submit proofs. Users don't reveal their Starknet address to the sequencer.
  • ✓
    Withdrawal timelock
    Configurable delay between deposit and withdrawal prevents timing correlation.
  • ✓
    Emergency withdraw
    Owner-initiated with 30-day timelock. Cancellable. Protects locked funds.
