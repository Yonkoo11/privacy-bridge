Privacy Bridge Flow EVM → Starknet

Deposit on one chain.
Withdraw on another.
No link between them.

Groth16 proofs sever the on-chain trail between Flow EVM deposits and Starknet withdrawals. Fixed denomination pools, Poseidon Merkle tree, relayer pattern.

  • TREE Poseidon depth-24
  • CALLDATA 1,977 felts

Protocol Flow
01
Lock

Deposit a fixed denomination into the PrivacyBridge contract on Flow EVM. A Poseidon commitment is inserted into the on-chain Merkle tree. The deposit amount and commitment are the only data emitted.

02
Prove

Generate a Groth16 proof in the browser. The proof attests to knowledge of a valid commitment in the tree without revealing which leaf it corresponds to. The secret and nullifier never leave the client.

03
Verify

The garaga verifier on Starknet checks the Groth16 proof on-chain. The nullifier hash is compared against the spent set. If the proof is valid and the nullifier is fresh, verification passes.

04
Claim

Shielded tokens are minted to the recipient address on Starknet. The nullifier is marked spent. No on-chain data connects this withdrawal to the original deposit on Flow EVM.

Security Properties
Fixed Denominations
Four pool sizes (0.0001, 0.001, 0.01, 0.1 FLOW). Uniform amounts prevent correlation by value.
On-chain Merkle Tree
Poseidon incremental tree, depth 24, with a 30-root history buffer. Inclusion proofs are verified against any recent root.
Commitment Hiding
Commitment = Poseidon(Poseidon(secret, nullifier), amount). The nested hash prevents amount leakage from the commitment alone.
Clean Events
Mint events emit nullifier_hash and amount only. No sender address, no deposit reference, no cross-chain identifiers.
Relayer Pattern
Withdrawals are submitted through a relayer so the recipient never has to fund a transaction from a linked address. Fee is bounded by max_fee_bps.
Withdrawal Timelock
Configurable delay between root relay and withdrawal eligibility. Prevents front-running of Merkle root updates.
Emergency Withdraw
Owner-initiated with a 30-day timelock. Provides a recovery path if the bridge or relayer becomes unavailable.

Known Limitations
Root relay is centralized

The watcher service relays Merkle roots from Flow EVM to Starknet using an owner key. A compromised key can delay but not forge roots.

Emergency withdraw is a single-key operation

Mitigated by a 30-day timelock. Users have a window to exit before funds can be moved. A multisig upgrade would remove this trust assumption.

Trusted setup is a 1-party ceremony

The Groth16 circuit uses a single-party powers-of-tau ceremony. A production deployment would require a multi-party ceremony to ensure the toxic waste is destroyed.

known_roots is unbounded on Starknet

The root history array grows without bound. Pruning was intentionally omitted because removing a root would lock any unspent deposits proven against it.
