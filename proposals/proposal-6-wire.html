Privacy Bridge
Flow EVM → Starknet

Deposit on one chain.
Withdraw on another.
No link between them.

Privacy Bridge uses Groth16 zero-knowledge proofs to break the on-chain link between deposits on Flow EVM and withdrawals on Starknet. The proof attests you know a valid commitment without revealing which one.

Proof system Groth16 / BN254
Protocol Flow
01
Lock
Deposit a fixed denomination into the PrivacyBridge contract. A Poseidon commitment is inserted into the on-chain Merkle tree. No link between your address and the commitment is exposed.
Flow EVM
02
Prove
Generate a Groth16 proof off-chain. The proof attests to knowledge of a valid leaf in the Merkle tree without revealing which one. 6,634 constraints.
Client-side
03
Relay
The current Merkle root is relayed from Flow to Starknet. A ring buffer of 30 roots ensures users have time to submit proofs against recent state.
Cross-chain
04
Claim
Submit the proof to the Starknet bridge contract. The garaga verifier checks the BN254 pairing on-chain. Funds mint to the recipient. Nullifier marked spent.
Starknet
Security Properties
Fixed denominations 0.0001, 0.001, 0.01, 0.1 FLOW. Equal amounts prevent value-based correlation.
On-chain Merkle tree Poseidon incremental, depth 24, 30-root ring buffer prevents front-running.
Commitment hiding Poseidon(Poseidon(secret, nullifier), amount). Preimage never published.
Clean events Mint emits nullifier_hash + amount only. No from/to. No CID references.
Relayer pattern Fee-protected via max_fee_bps. Relayer cannot extract more than user agreed.
Withdrawal timelock Configurable delay via root_timestamps. Grows anonymity set before claim.
Emergency withdraw 30-day timelock, owner-initiated. Users can exit via normal proof path.

Known Limitations
root_relay Centralized relayer pushes Merkle roots from Flow to Starknet. Can cause liveness failure but cannot steal funds.
trusted_setup Single-party ceremony. A compromised setup would allow forged proofs. Production would require multi-party computation.
anonymity_set Starts at zero. Privacy grows with each deposit into the same denomination pool. Dashboard warns users of current set size.
emergency_key Single owner key can trigger emergency withdrawal after 30-day timelock. Mitigated but not eliminated.
